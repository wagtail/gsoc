## Research
A demo website redesign is a prototype or simulation of a new website design, created to visualize, test, and approve changes before applying them to the live, public-facing site. It is a "working draft" or a staged version or a completely revamped website that allows stakeholders to see how new layouts, content, and branding will look and function without risking the current site's performance. 

Key Components of a Demo Redesign
Prototype/Staging Environment: The new design is built in a non-public "staging" or "local" environment (also called as subdomain).
Visual Mockups: It often starts as high-fidelity mockups in tools showing new color schemes, fonts, imagery, and layout structures.
Functional Testing: The demo allows developers to test interactive elements (buttons, forms, navigation) to ensure they work before the official launch.
Content Restructuring: It demonstrates a new information architecture, such as a rearranged navigation menu or new, updated website copy.
